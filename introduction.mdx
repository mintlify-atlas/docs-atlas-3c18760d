# Flutter Billing App

A feature-rich, high-performance **offline-first** billing and Point of Sale (POS) application built with Flutter. Designed for seamless retail checkout operations featuring barcode scanning, thermal Bluetooth printing, and robust local data persistence.

## What is Flutter Billing App?

Flutter Billing App is a complete offline POS system for small to medium-sized retail shops. It streamlines the checkout process, catalog management, and receipt generation - all running entirely on-device with no internet connection required.

<Note>
  This application is built using **Clean Architecture** principles with feature-driven design, ensuring scalability, separation of concerns, and robust testability.
</Note>

## Core Capabilities

### Product Management System

Manage your complete inventory with support for:
- Product CRUD operations (Create, Read, Update, Delete)
- Barcode and QR code scanning integration
- Stock level tracking
- Price management
- Unique product identification

### Smart Checkout System

Streamline your billing process with:
- Camera-based barcode scanning using `mobile_scanner`
- Manual product entry for items without barcodes
- Real-time cart total calculation
- Tax computation support
- Order finalization and receipt generation

### Bluetooth Thermal Printing

Generate professional receipts with:
- Direct Bluetooth connection to thermal POS printers
- Customizable shop header (name, address, contact details)
- Itemized product listing with prices
- Tax and total calculations
- Instant printing via `print_bluetooth_thermal`

### Offline-First Architecture

Operate confidently without internet:
- **Hive** NoSQL local database for all data persistence
- Lightning-fast data access and queries
- No network dependency for core operations
- Perfect for exhibitions, markets, and areas with poor connectivity

## Use Cases

### Retail Checkout

A cashier launches the app, navigates to the checkout page, and uses the device camera to instantly scan product barcodes. The products are added to the cart, the total is calculated including taxes, and a receipt is finalized.

### Physical Receipt Generation

After checkout confirmation, the app triggers a connected external Bluetooth thermal POS printer to instantly print an itemized paper receipt with the shop's header information.

### Inventory Management

A manager opens the Product feature to add new stock to the local database, taking a picture of the barcode to bind the SKU for future lightning-fast checkouts.

### No-Connection Operation

The business operates a stall at an exhibition with poor networking. The app functions entirely via its embedded Hive local database and Bluetooth, completely undisturbed by network drops.

## Architecture Overview

The application follows **Feature-First Clean Architecture** with strict separation of concerns:

```text
lib/
├── core/                       # Core application utilities
│   ├── data/                   # Global data sources (Hive initialization)
│   ├── error/                  # Failure/Exception models (fpdart compatible)
│   ├── theme/                  # UI styling and typography
│   ├── usecase/                # Base UseCase contracts
│   ├── utils/                  # Helpers (PrinterHelper, formatters)
│   └── widgets/                # Reusable UI widgets
│
└── features/                   # Independent feature modules
    ├── billing/                # POS operations: Cart, Checkout, Invoices
    ├── product/                # Inventory: Adding, Listing, Scanning
    ├── settings/               # Printer connections, App settings
    └── shop/                   # Shop details configuration
```

<Warning>
  Each feature is subdivided into Clean Architecture layers: `data`, `domain`, and `presentation`. Maintain strict boundaries between these layers when extending the application.
</Warning>
