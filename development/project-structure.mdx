## Clean Architecture Layers

### Domain Layer (Business Logic)
- **Entities**: Pure Dart classes representing business objects
- **Repositories**: Abstract interfaces defining data operations
- **Use Cases**: Single-responsibility business logic units
- Returns `Either<Failure, Result>` for functional error handling

### Data Layer (Data Sources)
- **Models**: Extend entities, add serialization annotations
- **Repository Implementations**: Concrete implementations using Hive
- Uses code generation for Hive type adapters (`.g.dart` files)

### Presentation Layer (UI)
- **BLoC**: State management with flutter_bloc
- **Pages**: UI screens and widgets
- Depends only on domain layer (entities and use cases)
