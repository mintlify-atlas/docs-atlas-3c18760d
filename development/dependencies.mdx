This page documents all the dependencies in the Flutter Billing App, extracted from `pubspec.yaml`, organized by category with explanations of their purpose.

## State Management & Dependency Injection

<Accordion title="flutter_bloc ^8.1.5">

Flutter widgets that make it easy to implement the BLoC (Business Logic Component) design pattern.

**Usage in the app:**
- State management for all features (Product, Shop, Billing, Printer)
- Separates business logic from UI
- Examples: `ProductBloc`, `ShopBloc`, `BillingBloc`, `PrinterBloc`

**Key files:**
- `lib/features/*/presentation/bloc/` directories contain all BLoC implementations

</Accordion>

<Accordion title="bloc ^8.1.4">

Core BLoC package (dependency of flutter_bloc). Provides the base BLoC classes and stream transformers.

**Usage in the app:**
- Foundation for flutter_bloc
- Event handling and state emission logic

</Accordion>

<Accordion title="get_it ^7.6.7">

Simple service locator for dependency injection.

**Usage in the app:**
- Centralized dependency injection container
- Registers repositories, use cases, and BLoCs
- Configured in `lib/core/service_locator.dart`

**Example registration:**
```dart
sl.registerFactory(() => ProductBloc(
  getProductsUseCase: sl(),
  addProductUseCase: sl(),
));
```

</Accordion>

<Accordion title="equatable ^2.0.5">

Simplifies equality comparisons for Dart classes.

**Usage in the app:**
- BLoC states and events extend Equatable
- Enables efficient state comparison in BLoC
- Reduces boilerplate for `==` operator and `hashCode`

</Accordion>

## Functional Programming

<Accordion title="fpdart ^1.1.0">

Functional programming in Dart and Flutter. Provides `Either`, `Option`, and other functional types.

**Usage in the app:**
- Error handling with `Either<Failure, Result>`
- All use cases return `Either<Failure, T>` for type-safe error handling
- Defined in base `UseCase` class: `lib/core/usecase/usecase.dart`

**Example:**
```dart
abstract class UseCase<Result, Params> {
  Future<Either<Failure, Result>> call(Params params);
}
```

</Accordion>

Declarative routing package for Flutter.

</Accordion>

## Database & Persistence

<Accordion title="hive ^2.2.3">

Lightweight and fast NoSQL database for Flutter.

**Usage in the app:**
- Primary local database for products, shop details
- Stores data in type-safe boxes
- No SQL required - uses type adapters for serialization

**Stored data:**
- Products (with barcode, price, stock)
- Shop information

</Accordion>

<Accordion title="hive_flutter ^1.1.0">

Flutter-specific extensions for Hive.

**Usage in the app:**
- Initializes Hive with Flutter-specific path
- Provides Flutter integration
- Configured in `lib/core/data/hive_database.dart`

</Accordion>

## Code Generation

<Accordion title="json_annotation ^4.9.0">

Annotations for json_serializable code generator.

**Usage in the app:**
- Annotates data models for JSON serialization
- Used with `@JsonSerializable()` annotation
- Generates `.g.dart` files with serialization code

</Accordion>

<Accordion title="build_runner ^2.4.8 (dev)">

Code generation tool runner.

**Usage in the app:**
- Runs code generators (Hive, JSON)
- Command: `dart run build_runner build --delete-conflicting-outputs`
- See [Code Generation](/development/code-generation) for details

</Accordion>

<Accordion title="json_serializable ^6.7.1 (dev)">

Generates JSON serialization code.

**Usage in the app:**
- Generates `fromJson()` and `toJson()` methods
- Works with json_annotation
- Creates `.g.dart` files

</Accordion>

<Accordion title="hive_generator ^2.0.1 (dev)">

Generates Hive type adapters.

**Usage in the app:**
- Generates type adapters for Hive models
- Annotated with `@HiveType()` and `@HiveField()`
- Used in `ProductModel`, `ShopModel`

**Example:**
```dart
@HiveType(typeId: 0)
class ProductModel extends Product {
  @HiveField(0)
  final String id;
  // ...
}
```

</Accordion>
