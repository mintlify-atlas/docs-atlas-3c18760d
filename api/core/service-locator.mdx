## Overview

The Service Locator provides centralized dependency injection management using the [GetIt](https://pub.dev/packages/get_it) package. It registers all application dependencies including BLoCs, use cases, and repositories, making them accessible throughout the app.

## Setup

### Initialization

Initialize the service locator before running the app:

```dart
import 'package:flutter_billing_app/core/service_locator.dart' as di;

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await di.init();
  runApp(MyApp());
}
```

## Registration Methods

### registerFactory

Registers a factory that creates a new instance every time it's called. Ideal for BLoCs and other stateful objects.

```dart
sl.registerFactory(
  () => ProductBloc(
    getProductsUseCase: sl(),
    addProductUseCase: sl(),
    updateProductUseCase: sl(),
    deleteProductUseCase: sl(),
  ),
);
```

### registerLazySingleton

Registers a singleton that's created only when first accessed. Perfect for use cases and repositories.

```dart
sl.registerLazySingleton(() => GetProductsUseCase(sl()));
sl.registerLazySingleton<ProductRepository>(
  () => ProductRepositoryImpl(),
);
```

## Accessing Dependencies

### Using sl()

Access any registered dependency using the generic `sl<T>()` syntax:

```dart
// Get a ProductBloc instance
final productBloc = sl<ProductBloc>();

// Get a use case
final getProductsUseCase = sl<GetProductsUseCase>();

// Get a repository
final productRepository = sl<ProductRepository>();
```

### In BLoC Providers

```dart
BlocProvider(
  create: (_) => sl<ProductBloc>(),
  child: ProductScreen(),
)
```

## Architecture Pattern

The service locator follows Clean Architecture principles:

```
BLoC → Use Case → Repository → Data Source (Hive)
```

- **BLoCs**: Presentation layer, registered as factories
- **Use Cases**: Domain layer, registered as lazy singletons
- **Repositories**: Data layer interfaces, registered as lazy singletons
