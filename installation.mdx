# Installation

This guide will walk you through setting up the Flutter Billing App development environment and installing all necessary dependencies.

## Prerequisites

Before you begin, ensure you have the following installed on your system:

### Flutter SDK

<Note>
  **Required Version**: Flutter SDK `>=3.1.0 <4.0.0`
</Note>

Verify your Flutter installation:

```bash
flutter --version
```

If you need to install or upgrade Flutter, visit the [official Flutter installation guide](https://docs.flutter.dev/get-started/install).

### Development Environment

You'll need one of the following:

- **Android Studio** - For Android development and emulators
- **Xcode** - For iOS development (macOS only)
- **VS Code** - Lightweight alternative with Flutter extensions

### Optional Hardware

For testing the complete feature set:

- Physical Android/iOS device for testing on real hardware
- Bluetooth thermal POS printer for receipt printing functionality

<Warning>
  Barcode scanning and Bluetooth thermal printing features require a physical device or appropriate hardware. These features will not work on emulators/simulators.
</Warning>

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the Flutter Billing App repository to your local machine:

    ```bash
    git clone <repository_url>
    cd billing_app
    ```

    Replace `<repository_url>` with the actual Git repository URL.
  </Step>

  <Step title="Install Dependencies">
    Fetch all required Flutter packages defined in `pubspec.yaml`:

    ```bash
    flutter pub get
    ```

    This will install all dependencies including:
    - **State Management**: `flutter_bloc`, `get_it`, `equatable`
    - **Database**: `hive`, `hive_flutter`
    - **Navigation**: `go_router`
    - **Hardware**: `mobile_scanner`, `print_bluetooth_thermal`
    - **Utilities**: `fpdart`, `intl`, `google_fonts`
    - And more...
  </Step>

  <Step title="Run Code Generation">
    Generate required code for Hive adapters and JSON serialization:

    ```bash
    dart run build_runner build --delete-conflicting-outputs
    ```

    <Note>
      This step is **crucial** and must be run before launching the app. It generates:
      - Hive type adapters for `ProductModel` and `ShopModel`
      - JSON serialization code for data models
    </Note>

    The `--delete-conflicting-outputs` flag ensures any existing generated files are replaced with fresh versions.
  </Step>

  <Step title="Verify Installation">
    Ensure everything is set up correctly:

    ```bash
    flutter doctor
    ```

    Check that all required dependencies are satisfied. Resolve any issues reported by Flutter Doctor.
  </Step>

  <Step title="Connect a Device or Start an Emulator">
    Check connected devices:

    ```bash
    flutter devices
    ```

    If no devices are listed:
    - **Android**: Start an Android emulator from Android Studio
    - **iOS**: Start an iOS simulator with `open -a Simulator`
    - **Physical Device**: Connect via USB and enable USB debugging
  </Step>
</Steps>

## Key Dependencies Explained

### State Management & Architecture

```yaml pubspec.yaml
# State Management & DI
flutter_bloc: ^8.1.5      # BLoC pattern for state management
get_it: ^7.6.7            # Service locator for dependency injection
equatable: ^2.0.5         # Value equality for immutable states
```

These packages implement the Clean Architecture pattern:
- **flutter_bloc**: Manages application state with the BLoC pattern
- **get_it**: Provides dependency injection via service locator
- **equatable**: Ensures immutable state comparisons in BLoCs

### Database & Persistence

```yaml pubspec.yaml
# Database
hive: ^2.2.3              # NoSQL local database
hive_flutter: ^1.1.0      # Flutter-specific Hive extensions
```

Hive provides:
- Lightning-fast local data storage
- Type-safe data models with adapters
- Completely offline operation
- No SQL queries - pure Dart objects

### Code Generation

```yaml pubspec.yaml
dev_dependencies:
  build_runner: ^2.4.8
  json_serializable: ^6.7.1
  hive_generator: ^2.0.1
```

These dev dependencies generate:
- `*.g.dart` files for Hive type adapters
- JSON serialization/deserialization code
- Boilerplate-free data models

### Hardware Integration

```yaml pubspec.yaml
# Scanner & Printer
mobile_scanner: ^5.1.0            # Camera-based barcode scanning
print_bluetooth_thermal: ^1.1.2   # Bluetooth thermal printer support
```

Enables:
- Real-time barcode/QR code scanning
- Direct connection to Bluetooth thermal printers
- Instant receipt printing

## Permissions Configuration

### Android Permissions

The app requires the following permissions (already configured in `AndroidManifest.xml`):

- **Camera**: For barcode scanning
- **Bluetooth**: For thermal printer connections
- **Storage**: For local database access

### iOS Permissions

Ensure your `Info.plist` includes:

- `NSCameraUsageDescription`: Camera access for barcode scanning
- `NSBluetoothAlwaysUsageDescription`: Bluetooth for thermal printing

## Troubleshooting

### Code Generation Issues

If you encounter errors during code generation:

```bash
# Clean generated files and rebuild
flutter clean
flutter pub get
dart run build_runner clean
dart run build_runner build --delete-conflicting-outputs
```

### Dependency Conflicts

If dependencies fail to resolve:

```bash
# Update all dependencies to latest compatible versions
flutter pub upgrade
```

### Build Failures

For build issues:

```bash
# Clean build cache and rebuild
flutter clean
flutter pub get
flutter run
```

<Warning>
  Always run `dart run build_runner build --delete-conflicting-outputs` after pulling new code that modifies data models. Missing generated files will cause runtime errors.
</Warning>
