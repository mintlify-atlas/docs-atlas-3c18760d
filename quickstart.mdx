This guide will walk you through setting up and running the Flutter Billing App on your local machine.

## Quick Launch

<Steps>
  <Step title="Clone the Repository">
    Clone the Flutter Billing App repository to your local machine:
    
    ```bash
    git clone https://github.com/Dinesh-Sowndar/flutter_billing_app.git
    cd flutter_billing_app
    ```
  </Step>

  <Step title="Install Dependencies">
    Fetch all required Flutter packages:
    
    ```bash
    flutter pub get
    ```
    
    This will install all dependencies specified in `pubspec.yaml`, including:
    - `flutter_bloc` for state management
    - `hive` and `hive_flutter` for local database
    - `mobile_scanner` for barcode scanning
    - `print_bluetooth_thermal` for thermal printing
  </Step>

  <Step title="Generate Code">
    Run code generation for Hive adapters and JSON serialization:
    
    ```bash
    dart run build_runner build --delete-conflicting-outputs
    ```
    
    This generates the `.g.dart` files required for:
    - Hive type adapters (`ProductModel`, `ShopModel`)
    - JSON serialization for data models
  </Step>

  <Step title="Run the App">
    Launch the app on your connected device or emulator:
    
    ```bash
    flutter run
    ```
    
    <Note>
    For the best experience testing barcode scanning and Bluetooth printing, use a physical Android or iOS device.
    </Note>
  </Step>
</Steps>

## Understanding the Initialization

When the app starts, it performs several initialization steps in `main.dart`:

```dart lib/main.dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await HiveDatabase.init();
  await di.init();
  runApp(const MyApp());
}
```

### Initialization Breakdown

<Accordion title="1. HiveDatabase.init()">
Initializes the Hive local database:

```dart lib/core/data/hive_database.dart
static Future<void> init() async {
  await Hive.initFlutter();
  
  // Register type adapters
  Hive.registerAdapter(ProductModelAdapter());
  Hive.registerAdapter(ShopModelAdapter());
  
  // Open boxes
  await Hive.openBox<ProductModel>('products');
  await Hive.openBox<ShopModel>('shop');
  await Hive.openBox('settings');
}
```

This sets up three Hive boxes:
- **products**: Stores all inventory items
- **shop**: Stores shop configuration (name, address, UPI ID)
- **settings**: Stores app settings (printer preferences, etc.)
</Accordion>

<Accordion title="2. di.init()">
Configures dependency injection with GetIt:

```dart lib/core/service_locator.dart
Future<void> init() async {
  // Register BLoCs as factories
  sl.registerFactory(() => ProductBloc(
    getProductsUseCase: sl(),
    addProductUseCase: sl(),
    updateProductUseCase: sl(),
    deleteProductUseCase: sl(),
  ));
  
  // Register use cases as lazy singletons
  sl.registerLazySingleton(() => GetProductsUseCase(sl()));
  sl.registerLazySingleton(() => AddProductUseCase(sl()));
  
  // Register repositories
  sl.registerLazySingleton<ProductRepository>(
    () => ProductRepositoryImpl(),
  );
}
```

This registers all the BLoCs, use cases, and repositories needed for Clean Architecture.
</Accordion>

<Accordion title="3. MultiBlocProvider Setup">
Provides BLoCs to the widget tree:

```dart lib/main.dart
MultiBlocProvider(
  providers: [
    BlocProvider<ProductBloc>(
      create: (context) => di.sl<ProductBloc>()..add(LoadProducts())
    ),
    BlocProvider<ShopBloc>(
      create: (context) => di.sl<ShopBloc>()..add(LoadShopEvent())
    ),
    BlocProvider<BillingBloc>(
      create: (context) => BillingBloc(getProductByBarcodeUseCase: di.sl())
    ),
    BlocProvider<PrinterBloc>(
      create: (context) => di.sl<PrinterBloc>()..add(InitPrinterEvent())
    ),
  ],
  child: MaterialApp.router(...)
)
```

Each BLoC is created and immediately dispatches its initial load event.
</Accordion>

## First Run Experience

When you first launch the app:

1. **Home Screen**: You'll see the barcode scanner interface with an empty cart
2. **Add Products**: Navigate to the Products tab to add items to your inventory
3. **Scan Barcodes**: Use the camera to scan product barcodes for quick checkout
4. **Configure Shop**: Set up your shop details in Settings for receipt printing

## Development Workflow

### Hot Reload

Flutter's hot reload makes development fast:

```bash
# Press 'r' in the terminal to hot reload
# Press 'R' to hot restart (full restart)
```

### Debugging

Enable debug mode for detailed logs:

```bash
flutter run --debug
```

### Code Generation Watch Mode

For continuous code generation during development:

```bash
dart run build_runner watch --delete-conflicting-outputs
```

This automatically regenerates `.g.dart` files when you modify model classes.

## Testing on Physical Devices

<Tabs>
  <Tab title="Android">
    1. Enable Developer Options on your Android device
    2. Enable USB Debugging in Developer Options
    3. Connect your device via USB
    4. Run `flutter devices` to verify connection
    5. Run `flutter run` to install and launch

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build Runner Fails">
    If code generation fails, try cleaning first:
    
    ```bash
    dart run build_runner clean
    dart run build_runner build --delete-conflicting-outputs
    ```
  </Accordion>

  <Accordion title="Hive Box Not Found">
    Ensure you've run code generation and HiveDatabase.init() is called:
    
    ```bash
    # Generate adapters
    dart run build_runner build --delete-conflicting-outputs
    
    # Clear app data and reinstall
    flutter clean
    flutter pub get
    flutter run
    ```
  </Accordion>

  <Accordion title="Camera Permission Denied">
    Add camera permissions to your manifest:
    
    **Android** (`android/app/src/main/AndroidManifest.xml`):
    ```xml
    <uses-permission android:name="android.permission.CAMERA"/>
    ```
    
    **iOS** (`ios/Runner/Info.plist`):
    ```xml
    <key>NSCameraUsageDescription</key>
    <string>This app needs camera access to scan barcodes</string>
    ```
  </Accordion>

  <Accordion title="Bluetooth Permission Issues">
    For Android 12+ (API level 31+), additional permissions are required:
    
    ```xml android/app/src/main/AndroidManifest.xml
    <uses-permission android:name="android.permission.BLUETOOTH_CONNECT"/>
    <uses-permission android:name="android.permission.BLUETOOTH_SCAN"/>
    ```
  </Accordion>
</AccordionGroup>
